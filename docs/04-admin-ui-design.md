# 04 - 管理者画面設計

## 1. 概要

管理者画面は **Admin（管理者 = Daisuke さん）** と **SE（ソリューションエンジニア）** 専用のダッシュボード。  
Zero Trust Access で保護され、`@cloudflare.com` ドメイン + グループメンバーシップで認証。

---

## 2. ロール別権限マトリクス

| 機能 | Admin | SE | User |
|---|:---:|:---:|:---:|
| ダッシュボード閲覧 | ✅ | ✅ | ❌ |
| 全記事の閲覧（下書き含む、読み取り専用） | ✅ | ❌ | ❌ |
| 自分の記事の編集・公開 | ✅ | ✅ | ✅ |
| ユーザー管理 | ✅ | ❌ | ❌ |
| ロール変更 | ✅ | ❌ | ❌ |
| Q&A 管理 | ✅ | ✅（担当記事のみ） | ❌ |
| テンプレートから AI ドラフト生成 | ✅ | ✅ | ✅ |

> **注:** 承認プロセスは廃止済み。全ユーザーが自分の記事を直接公開可能。管理者は全下書きを読み取り専用で閲覧できます。

| 🔴 | スパマー | spam@test.com | User | 0 | 0 | ⏳ 承認必須 (残3回) | 1週間前 | Blocked | [解除][削除] |

> **承認自動解除ルール:** User ロールのユーザーは、Admin が記事を **3回承認** した時点で、以降の投稿は承認スキップで即時公開されます。
> Admin/SE ロールは最初から承認不要です。Admin は管理画面からユーザーの `approved_post_count` を手動リセットすることも可能です。

**招待フロー:**
1. Admin が招待メールを送信（メールアドレス + 初期ロール指定）
2. ユーザーが招待リンクからアカウント作成
3. 自動的に指定ロールが付与
